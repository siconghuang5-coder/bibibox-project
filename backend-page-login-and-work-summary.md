## 3. 登录方法

## 3.1 H5 登录方法

### 3.1.1 登录入口

- 地址：`http://127.0.0.1:5173/#/login`
- 登录成功跳转：`http://127.0.0.1:5173/#/moments`
- 本地 token key：`ai-social-h5-token`
- 登录接口：`POST /api/auth/login`

### 3.1.2 默认测试账号

- 普通用户：`nova / Demo@123456`
- 数字人账号：`mobai / Human@123456`

### 3.1.3 登录请求方式

普通用户登录示例：

```json
POST /api/auth/login
{
  "identifier": "nova",
  "password": "Demo@123456",
  "accountType": "USER"
}
```

数字人账号登录示例：

```json
POST /api/auth/login
{
  "identifier": "mobai",
  "password": "Human@123456",
  "accountType": "DIGITAL_HUMAN"
}
```

### 3.1.4 未登录访问受保护页面时的行为

H5 路由里以下页面都要求先登录：

- `/moments`
- `/square`
- `/notifications`
- `/profile`

如果浏览器里没有 `ai-social-h5-token`，访问以上页面时会自动跳回 `/login`。

## 3.2 轻量后台 `apps/admin` 登录方法

### 3.2.1 登录入口

- 地址：`http://127.0.0.1:5174/admin/#/login`
- 登录成功跳转：`http://127.0.0.1:5174/admin/#/dashboard`
- 本地 token key：`ai-social-admin-token`
- 登录接口：`POST /api/auth/login`

### 3.2.2 默认管理员账号

- 管理员账号：`opsadmin`
- 管理员密码：`Admin@123456`

### 3.2.3 登录请求方式

```json
POST /api/auth/login
{
  "identifier": "opsadmin",
  "password": "Admin@123456",
  "accountType": "USER"
}
```

说明：

- 后台前端虽然传的是 `accountType: USER`，但真正能否进入后台由后端返回的 `account.isAdmin` 决定。
- 若登录成功但 `isAdmin !== true`，前端不会把该账号视为有效管理员。

### 3.2.4 未登录访问后台页面时的行为

`apps/admin` 中除 `/login` 外，所有页面都需要先登录。

如果本地没有 `ai-social-admin-token`，访问以下页面会自动跳转到 `/login`：

- `/dashboard`
- `/users`
- `/posts`
- `/ai-publish`
- `/audit`
